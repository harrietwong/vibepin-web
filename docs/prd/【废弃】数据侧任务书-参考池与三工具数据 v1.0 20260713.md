## WS1 存量元数据回填（最高优先级，零爬取成本）

**目标：** eligible 池 1,948 行的 visual_format / human_presence / composition_type / text_overlay_level 已知率从 8-18% → **≥80%**。

- 用现有分类器（backfill_pin_categories.py 一族）对存量 eligible 行全量重跑视觉分类；
- 顺跑 image_quality_band 校准：当前"high"占比 0-9% 明显失真，抽 50 张人工对照后调阈值再全量重打；
- 双 PRD 直接受益：三工具 P1-2 的 Format 列（弃前端 inferPinFormat）、Pin Detail "Why it works" 规则、参考池场景/真人适配打分。

**验收：** 5 个 P0 类目 visual_format 已知率 ≥80%；quality band 三档分布合理（high 不再≈0）。

## WS3 爬虫改造（产出率与字段质量）

1. **损耗前置：** 爬取端即丢弃 非≈2:3 比例、低分辨率、无图行；水印/质量分类并入爬取管线一步完成（不再存完再说）；
2. **详情级抓取：** 进 pin 详情页取 title/description/alt（当前 title 有效率仅 26-41%，直接影响三工具"不编造标题"红线与参考池匹配词汇）；
3. **入库硬校验：** pinterest_url/source_url 必填（linkback 合规展示依赖）、pin_id 去重；
4. **产出率审计：** 抽查当前 90% 不合格行的损耗原因分布，目标产出率 9.5% → **≥20%**。
