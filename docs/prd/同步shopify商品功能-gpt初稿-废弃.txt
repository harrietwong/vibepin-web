VibePin Ecommerce Integration — Phase 1 Shopify MVP

==================================================
1. Core planning rule
==================================================

Before proposing any implementation:

Audit the current code and verify the real existing capabilities.

Do not assume that models, database tables, APIs, OAuth infrastructure, background jobs, product pickers, asset libraries, or persistence layers already exist.

Clearly separate:

- verified existing capability;
- reusable but incomplete capability;
- missing capability;
- assumption requiring verification.

Do not rewrite Create Pins.

Use the principle:

Extract and extend, do not rebuild.

==================================================
2. Product goal
==================================================

The Phase 1 user workflow should be:

Connect Shopify
→ complete OAuth installation
→ run initial product sync
→ products appear in a shared Product Library
→ search and filter products
→ select one product
→ choose one or more product images
→ create or enhance Pin Draft Cards
→ AI Copy uses product metadata as grounded context
→ user explicitly chooses “Use product link as destination”
→ edit
→ select Pinterest Board
→ Smart Schedule / Publish.

Important product rules:

- Pin Draft Card remains the core publishing object.
- A Product is reusable context and an AI input.
- Syncing a Product must not create a Pin Draft automatically.
- Connecting Shopify must not automatically schedule or publish anything.
- Selecting a Product must not silently overwrite the Pin Website URL.
- Product URL belongs to the Product.
- Website URL / Destination URL belongs to the Pin.
- Only an explicit “Use product link as destination” action may copy the Product URL into an empty Pin URL.
- Replacing an existing Pin destination must require confirmation.
- Product images used for AI Image generation must not overwrite the original Pin image.
- Pinterest remains the only real publishing destination.
