# VibePin Pinterest Creative Intelligence Layer PRD

**版本：** v0.1
**状态：** Draft for Fable Review
**范围：** Create Pins、AI Image、AI Copy、Pinterest Keywords、Reference、Quality Evaluation
**产品原则：** Pinterest-first · Pin-draft-first · Product-aware · Reference-aware · AI on demand

---

## 1. 项目背景

VibePin 当前已经具备较完整的 Create Pins 闭环：

```text
Upload / Product Image
→ Pin Draft Card
→ AI Copy / AI Image
→ Edit
→ Pinterest Board
→ Schedule / Publish
```

现阶段主要质量问题不是流程缺失，而是生成内容仍可能出现：

* 图片好看，但缺乏 Pinterest 保存价值；
* 画面过度商业化、棚拍化或 AI 感明显；
* 商品与使用场景匹配不足；
* 缺少真人生活分享感；
* AI Copy 偏向图片描述，而不是搜索和收藏导向；
* 高搜索关键词与具体商品、场景和视觉方向结合不足；
* 一次生成后直接展示结果，缺少自动质量判断和筛选。

当前产品方向是建立一层 **Pinterest Creative Intelligence Layer**，将产品理解、视觉模式、创意方向、参考匹配、关键词、生成和质量判断连接起来。

---

## 2. 产品定位

Pinterest Creative Intelligence Layer 不是独立的图片生成器，也不是新的 Create Pins 工作区。

它是 VibePin 现有创建流程中的智能决策层：

```text
Product / Image understanding
→ Pinterest pattern matching
→ Creative direction
→ Reference and keyword context
→ Controlled generation
→ Quality evaluation
→ Pin Draft Card
→ Publish feedback
```

核心价值：

> 根据用户的商品、图片、Board 和内容目标，自动推荐更适合 Pinterest 的视觉方向、关键词和生成策略，并优先输出更真实、更鲜活、更像创作者分享的结果。

---

## 3. 产品目标

### 3.1 核心目标

1. 自动理解上传图片或商品的类别、属性和使用场景。
2. 为每个商品推荐 3–5 个适合 Pinterest 的创意方向。
3. 将高收藏内容规律沉淀为结构化 Visual Pattern Library。
4. 将创意方向、参考图、关键词和商品信息共同用于 AI Image 和 AI Copy。
5. 降低生成图的 AI 感、棚拍感和无场景感。
6. 自动评估多张生成结果，并优先展示质量更高的图片。
7. 记录用户选择、修改和发布结果，为后续模型优化提供数据。

### 3.2 用户感知目标

用户不需要自己写复杂 Prompt。

用户看到的应该是：

```text
Recommended for this product

• Cozy real-home styling
• Product in use
• Small-space organization idea
• Creator-style flat lay
• Tutorial / how-to Pin
```

而不是一个空白 Prompt 输入框。

---

## 4. 非目标

本阶段不做：

* 重写 Create Pins；
* 重做 AI Image Drawer；
* 新增复杂的创意工作台；
* 自动发布未经用户确认的内容；
* 自动复制其他 Pinterest Pin；
* 永久缓存未经授权的 Pinterest 图片作为训练素材；
* 展示虚假的 Save Potential、Click Potential 或 Trend Score；
* 用大段参数和高级设置增加用户负担；
* 替代现有 Pin Draft Card 生命周期；
* 将创意方向变成新的 Draft 状态。

## 6. 核心用户流程

### 6.1 上传图片流程

```text
Upload image
→ 创建 Unscheduled Pin Draft
→ 后台分析图片
→ 识别类别、商品属性、颜色、风格和适用场景
→ 推荐 Creative Directions
→ 用户可直接编辑、生成 Copy 或生成 AI Image
```

### 6.2 Shopify 商品流程

```text
Select Shopify product
→ 选择商品图片
→ 创建或关联 Pin Draft
→ 使用商品标题、描述、类别、标签和图片作为上下文
→ 推荐 Creative Directions
→ AI Image / AI Copy
```

### 6.3 AI Image 流程

```text
Edit Pin
→ Generate AI Image
→ 当前图片成为 Product Image #1
→ 系统推荐 3–5 个 Creative Directions
→ 用户选择方向
→ 可添加 Style References
→ Generate N Pins
→ 系统生成多个候选
→ Quality Judge 评估和排序
→ 每张结果创建独立 Pin Draft Card
```

### 6.4 AI Copy 流程

```text
Generate copy
→ Image context
→ Product context
→ Selected creative direction
→ Board context
→ Relevant Pinterest keywords
→ Generate title / description / alt text
→ Quality Gate
→ 成功后写入 Draft
```

## 7. 功能需求

## 7.1 Product and Image Analysis

系统需要从用户图片或商品信息中提取：

* product category；
* subcategory；
* visible objects；
* materials；
* dominant colors；
* visual style；
* likely use cases；
* likely audience；
* product scale；
* packaging / label information；
* scene suitability；
* whether background removal or cleanup is needed；
* whether the image is suitable for lifestyle generation；
* OCR text；
* brand/product invariants。

示例输出：

```json
{
  "category": "home_storage",
  "productType": "wooden entryway cabinet",
  "materials": ["wood", "metal"],
  "colors": ["natural oak", "black"],
  "useCases": [
    "entryway organization",
    "small apartment storage",
    "living room styling"
  ],
  "recommendedSceneTypes": [
    "real home corner",
    "before and after",
    "small-space organization"
  ]
}
```

分析过程不能阻塞用户手动编辑 Draft。

---

## 7.2 Visual Pattern Library

建立结构化 Pinterest 视觉模式库。

每个 Pattern 至少包含：

| 字段                  | 含义             |
| ------------------- | -------------- |
| Pattern name        | 模式名称           |
| Suitable categories | 适用商品类别         |
| Scene type          | 场景类型           |
| Composition         | 构图规则           |
| Lighting            | 光线规则           |
| Human presence      | 是否需要真人、手部或使用动作 |
| Creator realism     | 真人分享感规则        |
| Text overlay        | 推荐文字叠加类型       |
| Search intent       | 对应搜索意图         |
| Keywords            | 推荐关键词组         |
| Prompt seed         | Prompt 基础结构    |
| Negative rules      | 禁止项            |
| Example references  | 有权使用的参考素材      |

初始 Pattern 示例：

```text
Cozy real-home corner
Product in use
Small-space before and after
Creator-style flat lay
Sunday reset
Vanity routine
Three ways to style
Tutorial / how-to
Gift-ready packaging
Searchable product roundup
```

Pattern Library 初期允许使用人工规则与大模型结构化生成共同维护。

## 7.3 Creative Direction Generator

系统根据以下信息推荐 3–5 个方向：

* image analysis；
* product category；
* product metadata；
* Pinterest Board；
* keywords；
* available references；
* user language；
* season or campaign context。

每个 Creative Direction 包含：

```text
Direction name
Why it fits
Scene
Composition
Lighting
Human presence
Creator style
Pinterest intent
Suggested keywords
Prompt instructions
Negative rules
```

示例：

```text
Small-Space Entryway Upgrade

Why it fits:
The cabinet is suitable for apartment storage and entryway organization.

Scene:
A real apartment entryway with shoes, keys and a small rug.

Composition:
Slightly off-center vertical phone-photo composition.

Lighting:
Soft natural window light.

Creator style:
Lived-in, useful and realistic rather than showroom-perfect.

Pinterest intent:
Small apartment organization and entryway ideas.
```

## 7.4 Creative Direction UI

Creative Directions 放在现有 **Generate AI Image Drawer** 内，不新增独立工作区。

建议位置：

```text
Product images
Style references
Recommended directions
Creative direction
Advanced controls
Generate N Pins
```

展示形式：

```text
Recommended for this product

[ Cozy real-home setup ]
[ Product in use ]
[ Small-space idea ]
[ Tutorial / how-to ]
```

规则：

* 默认显示 3–5 个；
* 用户只能选择一个主方向；
* 用户可以编辑方向；
* 用户可以重新生成方向；
* 系统需要说明推荐理由；
* 方向选择不能自动创建 Draft；
* 只有图片生成结果才创建新 Draft。

## 7.5 Reference Recommendation

系统未来支持根据商品图片推荐参考素材：

```text
Product image
→ image/text embedding
→ category and metadata filtering
→ reference retrieval
→ relevance ranking
→ Recommended for this product
```

参考素材优先来源：

* 用户自己上传的参考图；
* My References；
* 用户历史成功素材；
* 品牌自有图片；
* 有授权的 stock / creator references；
* 经确认允许展示的外部引用。

每张 Reference 需要保留：

* source；
* provenance；
* usage rights；
* category；
* scene；
* style；
* embedding；
* user-selected count；
* accepted / rejected feedback。

Pinterest 来源素材不能默认永久缓存或直接用于生成衍生内容，具体策略需经过数据与合规审查。

---

## 7.6 Pinterest Keyword Intelligence

关键词匹配继续遵循：

```text
Relevance first
Search volume second
```

上下文包括：

* image summary；
* product category；
* product title and tags；
* selected creative direction；
* Board；
* page context；
* season。

推荐 5–8 个关键词。

关键词分组建议：

```text
Primary search intent
Style modifiers
Use-case keywords
Problem / solution keywords
Long-tail keywords
Seasonal keywords
```

禁止：

* 只按搜索量排序；
* 无依据使用 Trending；
* 展示未实际使用的关键词；
* 将英文关键词伪装成本地化关键词。

## 7.7 AI Image Prompt Builder

AI Image Prompt 不再只使用简单的商品描述。

Prompt 结构：

```text

Creative goal
Scene direction
Composition
Lighting
Human presence

Pinterest intent
Reference context
Negative rules
```

必须尽量保持：

* 主色；
* Logo；
* 标签；
* 包装；
* 尺寸比例；
* 材质特征。

统一 Negative Rules 示例：


## 7.9 Quality Judge and Reranker

生成 N 张图片后，系统对候选结果进行评估。

基础评价维度：

| 维度                   | 说明                    |
| -------------------- | --------------------- |
| Product preservation | 商品是否被改形、改色或丢失细节       |
| Realism              | 是否像真实照片               |
| Creator likeness     | 是否具有自然分享感             |
| Scene compatibility  | 商品与场景是否合理             |
| Pinterest fit        | 是否具有搜索与保存价值           |
| Composition          | 是否适合 2:3 Pinterest 画面 |
| AI artifacts         | 手部、文字、反光、结构是否异常       |
| Safety               | 是否满足内容安全要求            |

行为：

```text
Generate 4 candidates
→ judge all candidates
→ rank candidates
→ hide only clearly invalid candidates
→ show best candidates first
```

系统不应只生成一张并立即认为成功。

用户仍然可以查看未被判定为无效的其他结果。

## 10. 系统约束

1. 不覆盖原始上传图。
2. 每张生成结果创建独立 Pin Draft Card。
3. Creative Direction 不是 Draft lifecycle。
4. Quality Judge 失败不能污染正常 Draft。
5. AI Copy 失败不能覆盖已有字段。
6. 所有用户选择和结果跟随账号体系服务端持久化。
7. localStorage 只能作为缓存或恢复机制。
8. 不展示内部 chain of thought。
9. 不将外部参考素材描述为“复制这个 Pin”。
10. 不将未经验证的高收藏规律包装成保证表现。

## 15. PRD 验收标准

PRD 对应功能完成后，应满足：

* 用户上传商品图后可以得到 3–5 个有理由的创意方向；
* 用户无需写复杂 Prompt；
* 选择方向后，AI Image Prompt 包含场景、构图、光线、真实感和禁止项；
* 生成图片保留真实商品关键特征；
* AI Copy 同时使用图片、商品、方向、关键词和 Board；
* 一次生成多个候选时，优质结果优先显示；
* 用户选择、拒绝和发布行为能够记录；
* 不覆盖原图；
* 不增加新的 Draft lifecycle；
* 不重写 Create Pins；
* 不引入未经授权的 Pinterest 图片训练或缓存流程；
* 所有数据跟随账号体系跨设备持久化。
