项目介绍：
VibePin 是一款面向 Pinterest 创作者、电商卖家、数字产品卖家的 AI 创作工具。核心目标是帮助用户从 Pinterest 市场趋势、Viral Pins、Shop Signals、Trend Radar 中发现机会，然后生成 Weekly Plan，并通过 Create Pin Studio 使用 AI 批量生成 Pinterest Pin 图片和文案。

VibePin 的核心流程包括：
1. Workspace：发现和选择值得做的 opportunity。
2. Trend Radar：关键词趋势和需求/竞争分析。
3. Viral Pins：高保存量 pin 的视觉信号发现。
4. Shop Signals：真实商品或 digital product pin/product signals。
5. Weekly Plan：本周内容生产计划和生成结果管理。
6. Create Pin Studio：根据 opportunity、reference pins、product images、market angle 和 instructions 生成 Pin 图片。
7. History / Generated Pins：查看生成历史、批量加入 Weekly Plan、下载和继续编辑。

一、已经确定的核心产品原则

1. Workspace 是机会发现页，不要强行改它。
Workspace 当前真实标签体系是：
- Primary Badge：
  - Best Bet：score >= 75，绿色 #16A34A
  - Steady：score 55–74，蓝色 #2563EB
  - Competitive：score < 55，橙色 #D97706
- Trend State：
  - ↑ Rising：YoY >= 100% 或 lifecycle = rising，绿色 #059669
  - ∞ Evergreen：默认稳定，蓝色 #0284C7
  - ◎ Seasonal：lifecycle = seasonal，紫色 #9333EA

Workspace 每张卡片头部最多两个 badge：
[Best Bet / Steady / Competitive] + [↑ Rising / ∞ Evergreen / ◎ Seasonal]

Workspace 不改。其他页面尽量向 Workspace 的评价体系靠齐。

2. 全站评价体系要简单。
每个页面最多 3 类评价词。
每张卡 / 每行最多显示 2 个 visible badges。
全站尽量统一：
- Primary Signal：Best Bet / Steady / Competitive
- Trend State：Rising / Evergreen / Seasonal
- 第三个状态如果需要，只能是操作状态，如 Pins ready、Product added、Not validated，不能再加一堆市场标签。

不要再主显示：
- Blue Ocean
- Early Trend
- Hot but Crowded
- Low Priority
- High Demand Hidden Supply
- New Account Friendly
- Over-saturated
- Low Volume

这些可以作为 View Evidence 里的解释文案或内部逻辑，不要作为主 badge。

3. 不要默认暴露数据库总量。
避免显示：
- total keywords
- total products
- linked pins count
- total saves
- category-level counts
- 128 products / 200 keywords 这种数字

允许显示：
- 用户当前选择数：2 references selected、3 products added
- 生成公式：2 references × 2 images each = 4 pins
- row-level 质量指标：High / Medium / Low
- 当前操作状态：4 pins ready

4. 页面分工：
- Trend Radar = 关键词需求验证中心，keyword-first，不要强依赖图片。
- Viral Pins = 真实高表现 pin 的视觉发现页，image-first。
- Shop Signals = 真实商品 / 外链 / digital-intent pin signal，image/product-first。
- Workspace = 机会卡片与批量选择入口。
- Weekly Plan = 周内容生产总览，显示每天计划、已生成图片、产品状态、下一步操作。
- Create Pin = 生成执行页。
- History = 最近生成记录和批量处理入口。
- 未来 Pin Library = 大量生成资产的长期管理页。

二、Create Pin Studio 已讨论过的关键决策

1. Create Pin 不应该必须从 Weekly Plan 进入。
主流程应支持：
Workspace → Create Pins
Workspace 多选 → Create Batch
Weekly Plan → Continue / Create Pins
Shop Signals → Create with selected products
Trend Radar → Create from keyword
Viral Pins → Use as reference / Create Pin
Create Pin 空状态 → 作为 start hub，支持 Trend Radar / Viral Pins / Weekly Plan / Scratch / History。

2. Create Pin 空状态：
如果用户直接打开 /app/studio，没有选择 opportunity，不要只显示空白。
应该显示：
- 左侧 Start from an Opportunity：Weekly Plan、Trend Radar、Viral Pins 最近项
- 中间大卡片：
  - From Trend Radar
  - From Viral Pins
  - From Scratch
- How it works
- History 按钮

3. Reference Pins 逻辑：
用户可以选择多张 reference pins。
但多张 reference 不应该混成一个模糊 prompt。
默认逻辑是：
一个 selected reference = 一个 style group。
如果选择 2 张 reference，images per reference = 2，则应该生成：
Reference 1 → 2 张
Reference 2 → 2 张
总共 4 张，并按 reference group 展示。

非常重要：
如果 UI 显示 “2 references × 2 images each = 4 pins”，实际生成结果也必须是 4 张。之前多次出现 UI 显示 4，但实际只生成 2 的 bug，这说明 generate handler 还停留在 single-reference mode。

正确生成逻辑：
for each selectedReference:
  call generate API with:
    - this reference image
    - all selected product images
    - imagesPerReference
    - keyword/category/brief
    - category-aware prompt
  store results grouped by reference

不要只用 selectedReferences[0]。

4. Reference 的角色：
Reference pin 只作为 style guide，不是要复制别人场景。
它用于：
- aesthetic direction
- mood
- color palette
- composition density
- lighting
- Pinterest-native visual language

不要：
- copy exact scene
- copy camera angle
- copy layout
- 直接把用户产品硬塞到原参考图场景里

产品优先：
如果上传产品不适合原 reference 场景，AI 应该允许重构场景，让产品自然融入，但保留 reference 的风格方向。
例如 Home Decor 中，产品可以被放入：
- styled shelf
- cozy corner
- room vignette
- interior moodboard
- bedroom setup
- living room setup

5. Product Images 逻辑：
支持多产品上传。
多个产品构成 product set。
同一个 product set 可以复用到多个 reference groups。
每个 output 应尽量自然包含大部分产品，但不要为了硬塞产品导致构图不合理。
产品可适配尺度、摆放方式、周边道具，但要保持 recognizable。

6. 生成图默认不要文字。
默认 strict no text overlay：
- No text overlay
- No English words
- No typography
- No labels
- No watermark
除非后续用户明确选择 Add text overlay。

7. Variations 这个词用户不懂。
应改为：
Images per reference
底部摘要推荐：
2 references × 2 images each = 4 pins · 2 products · No text
按钮：
Generate 4 Pins

不要写：
Generate 4 outputs in 2 style groups
这种用户不一定懂的内部语言。

三、Generated Pins / History / Weekly Plan 闭环

1. Generated Pins 必须变成 production assets，而不是临时图片缓存。
每张 generated pin 至少需要状态：
- generated
- added_to_plan
- downloaded
- published（如果没有真实发布功能，不要显示 fake published）

2. Add to Weekly Plan 后，Weekly Plan 必须能看到图片。
正确闭环：
Create Pin 生成图片
→ 用户 Add selected / Add group / Add all to Plan
→ pin.status = added_to_plan
→ weeklyPlanItemId 被写入
→ Weekly Plan 对应 row 显示 thumbnails
→ 行动按钮从 Create Pins 变成 View Pins / Create More
→ History 也显示 Added / Partially added 状态
→ 刷新后不丢

3. History Drawer 设计原则：
History Drawer = 快速批量处理最近生成结果。
它应该支持：
- 搜索
- 状态筛选：All / Not added / Added / Downloaded
- session card 多选
- 展开 session 后单张 pin 多选
- Add selected to Plan
- Download selected
- Add all to Plan
- Add remaining to Plan
- View in Plan
- Open

用户不应该必须进入 modal 才能把图加入 Plan。

4. History Modal 设计原则：
History Modal = 详细查看 session。
应该支持：
- reference group 展示
- 每张图 checkbox
- Add selected to Plan
- Add group to Plan
- Add all to Plan
- Download selected / all
- 每张卡显示状态：Generated / Added
- Load in Studio
- View in Plan

5. Weekly Plan 设计原则：
Weekly Plan = 内容生产总览，不是文字 brief 列表。
每条 row 应该显示：
- 日期
- title
- 2 个 badges：Best Bet/Steady/Competitive + Rising/Evergreen/Seasonal
- pin thumbnails
- pin count 状态：
  - 0 pins yet
  - 2 pins added
  - 4 pins ready
- product state：
  - No product
  - 2 products used
  - Product set used
- actions：
  - no pins → Create Pins
  - has pins → View Pins + Create More

空状态不要用大块 dashed box。
如果没图，用 3–4 个小 placeholder slots：
[+] [+] [+] [+]
0 pins yet

如果有图，显示 2:3 缩略图，最多 4 张，超过显示 +N。

四、Trend Radar 已讨论过的方向

1. Trend Radar 是 keyword-first，不应强依赖图。
目前因为数据库很多关键词没图，如果有的行有图、有的行没图，会显得不统一。
结论：
Trend Radar 主表不要显示 image thumbnails。
保持全表统一：
Keyword
Category
Signal
Trend
Demand
Competition
Actions

Keyword cell 可以有小 category icon，但不要有 thumbnail。
图片应该留给 Viral Pins / Shop Signals / Create Pin / Weekly Plan。

2. Trend Radar 的页面结构：
建议使用 Image 1 的整体表格清晰度，但不要图。
顶部使用 Physical Products / Digital Products segmented control。
在 mode 内部有：
Verified Trends / Seed Ideas

Physical Products：
- 显示真实 keyword trend rows
- category filters
- Signal filters：All / Best Bet / Steady / Competitive
- Trend filters：All / Rising / Evergreen / Seasonal
- Search
- Sort by Opportunity

Digital Products：
- Verified Trends：真实验证过的 digital keyword / signal-backed rows
- Seed Ideas：候选词库，明确 Not validated
- niche filters
- format filters
- actions：Check Trends / Create / Plan

Seed Ideas 必须明确：
Seed ideas are starting points, not market signals yet.
Check Pinterest Trends before planning or creating.

3. 评价体系：
Trend Radar 不要再主显示：
Blue Ocean / Early Trend / Hot but Crowded / Low Priority。
要改成和 Workspace 对齐：
- Signal：Best Bet / Steady / Competitive
- Trend：Rising / Evergreen / Seasonal

4. Digital Product 方向：
之前 Claude 把 Shop Signals Digital 改成了静态关键词表，这是错误方向。
已纠正：
- Shop Signals Digital 应该显示真实 digital-intent pins / products，不是 seed keyword table。
- Trend Radar 的 Digital Seeds 可以是静态候选词，但必须标注未验证。
- Digital Seeds 不等于市场信号。

Digital product discovery 正确流程：
真实 digital-intent pin signal
→ 抽取候选关键词
→ 去 Pinterest Trends 验证
→ 再 Create / Plan

五、Shop Signals 已讨论过的方向

1. Shop Signals = 真实商品/外链信号页。
Physical tab：
显示真实商品卡片。

Digital tab：
显示真实 digital-intent pin/product cards，而不是静态 keyword library。
可以判断 digital-intent 的 tokens：
printable, template, worksheet, planner, tracker, checklist, spreadsheet, notion, canva, editable, download, pdf, svg, png, clipart, digital paper, invitation, mockup, preset, ebook, guide, TPT, Teachers Pay Teachers, Payhip, Gumroad, Creative Market, Creative Fabrica

排除或降权：
bluetooth, speaker, phone case, case, cover, screen, lamp, chair, plant, dress, shoes, bag, necklace, toy, mug, shirt, jewelry, shipping, physical item

2. Shop Signals Digital card 应显示：
- real pin / product image
- title
- platform/source：Etsy / TPT / Payhip / Gumroad / Creative Market 等
- digital format badge：Printable / Template / Worksheet / Planner / Canva Template / PDF Guide
- qualitative signal，不要假数据
- actions：
  - View source
  - Use as reference
  - Create Pins
  - Extract keyword
  - Check Trends

3. Shop Signals 需要多选产品。
用户应该能选择多个 products → Create Pin 作为 product set。
每张卡支持 checkbox。
顶部或底部有：
3 products selected
[Create Pins with selected products] [Clear]
